## О проекте

Этот репозиторий содержит исследовательский Unity-проект, посвящённый автоматической генерации иерархических сетей задач (HTN) на основе данных, полученных в ходе обучения с подкреплением (RL).

Идея работы состоит в том, чтобы:
1. Обучить игрового агента в Unity с помощью **Unity ML-Agents**;
2. Собрать логи инференса обученной модели;
3. Выполнить предобработку данных;
4. Автоматически сформировать HTN-модель, которая воспроизводит поведение обученного агента.

Основная логика взаимодействия компонентов проекта подробно описана в магистерской диссертации по этой теме.

## Структура репозитория

В репозитории находятся основные скрипты для обработки данных и генерации HTN:

- **`Data preprocessing.py`** — предобработка логов, полученных после обучения агента с подкреплением
- **`HTN Generation.py`** — генерация иерархической сети задач (HTN) на основе обработанных данных

## Как начать работу

### 1. Установите Unity

Скачайте и установите Unity Hub и подходящую версию Unity Editor.

### 2. Скачайте проект из GitHub

Склонируйте репозиторий или скачайте архив с проектом и откройте его через Unity Hub.

### 3. Установите Unity ML-Agents

В проект необходимо добавить пакет **Unity ML-Agents**.

Официальная документация:
[Unity ML-Agents Toolkit](https://github.com/Unity-Technologies/ml-agents)

### 4. Настройте Unity ML-Agents

Базовый порядок настройки:

1. Откройте проект в Unity.
2. Через **Package Manager** добавьте пакет ML-Agents.
3. Убедитесь, что в проекте подключены необходимые зависимости Python-среды для обучения.
4. Настройте сцену, агентов, наблюдения, действия и систему вознаграждений.
5. Запустите обучение и соберите логи инференса для дальнейшей обработки.

Подробная настройка ML-Agents зависит от версии Unity и версии пакета, поэтому рекомендуется сверяться с официальной документацией.

## Алгоритм работы с проектом

1. **Создание среды в Unity**  
   Подготавливается игровая сцена, объекты, агенты, награды и условия завершения эпизода.

2. **Обучение агента с подкреплением**  
   Агент обучается в среде Unity с помощью ML-Agents.

3. **Сбор логов инференса**  
   После обучения сохраняются данные о состояниях, действиях и результатах поведения агента.

4. **Предобработка данных**  
   Скрипт `Data preprocessing.py` предобрабатывает данные, выводит результаты предобработки и формирует JSON-файл с готовой HTN структурой.

5. **Генерация HTN**  
   Скрипт `HTN Generation.py` строит иерархическую сеть задач на основе обработанных данных в виде C#-файлов.

6. **Интеграция в Unity**  
   Сгенерированная HTN-модель может быть использована внутри игрового проекта.

## Основные результаты

В рамках исследования была подтверждена гипотеза о том, что поведение, полученное в процессе обучения с подкреплением, можно использовать как источник для автоматического формирования интерпретируемой поведенческой модели.

Полученные результаты показали, что:

- автоматически сгенерированная HTN-модель сохраняет ключевую логику принятия решений обученного агента;
- подход демонстрирует сопоставимую эффективность с исходной RL-политикой;
- по сравнению с вручную написанной **rule-based** моделью автоматическая HTN показывает сопоставимые результаты по среднему счёту и устойчивости поведения;
- метод подходит для повторного использования, расширения и проведения новых экспериментов.
